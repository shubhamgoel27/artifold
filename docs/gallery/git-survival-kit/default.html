Git Survival Kit

The 14 essential git commands every new developer should know, with working examples.

git init

Create a new git repository in the current directory.

git init
git init my-project

git clone

Copy an existing remote repository to your local machine.

git clone https://github.com/user/repo.git
git clone git@github.com:user/repo.git my-folder

git status

Show which files are modified, staged, or untracked.

git status
git status -s   # short format

git add

Stage changes to be included in the next commit.

git add README.md
git add .              # stage everything
git add -p             # interactively pick hunks

git commit

Record staged changes to the repository history.

git commit -m "Add login form"
git commit -am "Quick fix"   # add tracked + commit
git commit --amend           # edit last commit

git log

View commit history with various formatting options.

git log
git log --oneline
git log --graph --oneline --all

git diff

Show changes between commits, branches, or working tree.

git diff               # unstaged changes
git diff --staged      # staged changes
git diff main feature  # compare branches

git branch

List, create, or delete branches.

git branch                 # list local branches
git branch new-feature     # create branch
git branch -d old-feature  # delete merged branch

git checkout / switch

Switch between branches or restore files.

git switch main
git switch -c new-feature   # create + switch
git checkout -- file.txt    # discard changes

git merge

Combine changes from another branch into the current one.

git switch main
git merge feature-branch
git merge --no-ff feature-branch

git pull

Fetch from remote and merge into the current branch.

git pull
git pull origin main
git pull --rebase

git push

Upload local commits to a remote repository.

git push
git push origin main
git push -u origin new-feature   # set upstream

git reset

Undo commits or unstage files. Use with care.

git reset HEAD file.txt     # unstage
git reset --soft HEAD~1     # undo last commit, keep changes
git reset --hard HEAD~1     # discard last commit

git stash

Temporarily shelve uncommitted changes so you can switch context.

git stash
git stash list
git stash pop

git remote

Manage the set of remote repositories tracked by your project.

git remote -v
git remote add origin https://github.com/user/repo.git
git remote remove origin
Pro tip: Run git config --global user.name "Your Name" and git config --global user.email "you@example.com" once when setting up a new machine, so your commits are attributed properly.
