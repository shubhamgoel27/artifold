Transformers, From an LSTM Person's Point of View

A working guide to self-attention, multi-head projection, positional encoding, and the full encoder–decoder stack.

If you have spent time wiring up LSTMs or stacking dilated 1-D convolutions for sequence tasks, the Transformer is going to feel both very simple and very strange. There is no recurrence. There is no convolution. The whole architecture is built out of one operation — scaled dot-product attention — applied in parallel across every position of the sequence. This piece walks through why the field abandoned recurrence, what attention actually computes, and how the pieces snap together into the encoder–decoder model from Vaswani et al. (2017).

1. Why RNNs hit a wall

LSTMs and GRUs solved the vanishing-gradient problem well enough to make sequence modeling practical, but they have two structural defects that get worse as models and datasets grow:

  • Sequential compute. The hidden state ht depends on ht-1. You cannot compute step t until step t-1 is done. On a modern GPU with thousands of cores, this leaves most of the silicon idle. Training time scales linearly with sequence length, and you cannot trade extra hardware for shorter wall-clock time.
  • Information bottleneck. Long-range dependencies must be funneled through a fixed-size hidden vector across many time steps. Even with gating, signal at position t reaching position t+500 is mediated by 500 nonlinear updates. Gradients still degrade, and the path length between any two tokens is O(n).

Convolutions partially fix the parallelism issue — WaveNet and ByteNet stack dilated convolutions to expose long-range structure — but the receptive field still grows only logarithmically with depth, and the model has no built-in way to route information based on content.

The Transformer's pitch: every position should be able to look at every other position in one step, and the lookup pattern should be learned from the data itself, not baked into the topology.

2. Scaled dot-product attention

Attention treats a sequence as a soft, content-addressable memory. Each position emits three vectors: a query (what am I looking for?), a key (what do I contain?), and a value (what should I hand over if you pick me?). To produce a new representation for a position, you compare its query against every key, normalize the scores into a probability distribution, and take the weighted sum of values.

Concretely, pack the queries, keys, and values for a whole sequence into matrices Q, K, V with shapes (n × dk), (n × dk), and (n × dv). The output of attention is:

Scaled Dot-Product Attention Attention(Q, K, V) = softmax ( Q KT / √dk ) V

Three things to internalize:

  1. It is a matrix multiply. QKT is one GEMM. Softmax is element-wise along rows. The final multiply against V is another GEMM. Two dense matmuls on a GPU are roughly free compared to a 500-step recurrence.
  2. The √dk scaling matters. Without it, when dk is large the dot products have variance ~dk, the softmax saturates, and gradients vanish. Dividing by √dk keeps the pre-softmax logits at unit scale.
  3. The path length is O(1). Any token can attend to any other in a single layer. Compare that to O(n) for an RNN or O(log n) for a dilated CNN.

Where Q, K, V come from

For a single input matrix X of shape (n × dmodel), they are three learned linear projections:

Q = X WQ   K = X WK   V = X WV

When Q, K, and V all come from the same sequence, this is self-attention. In the decoder, queries come from the decoder state while keys and values come from the encoder's output — this is cross-attention, and it is the only place the decoder gets to look at the source sequence.

3. Multi-head attention

A single attention operation has to compress every kind of relationship — syntactic, semantic, positional — into one weighted average. The fix is to run h attention operations in parallel, each on a lower-dimensional projection, then concatenate and project back:

Multi-Head Attention MultiHead(Q, K, V) = Concat(head1, …, headh) WO
where  headi = Attention(Q WiQ, K WiK, V WiV)

In the original paper, dmodel = 512 and h = 8, so each head works in a dk = dv = 64-dimensional subspace. The total compute is roughly the same as one full-width attention, but different heads learn to specialize — one might track subject–verb agreement, another might attend to the previous punctuation mark, and so on.

4. Positional encoding

Self-attention is a set operation. If you permute the input tokens, the outputs permute the same way. That is fatal for language, where "dog bites man" is not "man bites dog." Transformers fix this by adding a position-dependent vector to each token embedding before the first layer.

The original paper uses fixed sinusoids of geometrically spaced frequencies:

PE(pos, 2i) = sin (pos / 100002i/dmodel)
PE(pos, 2i+1) = cos (pos / 100002i/dmodel)

Two useful properties fall out of this choice. First, each dimension is a sinusoid of a different wavelength, so the model can read off position at multiple scales the way Fourier features do. Second, because sin and cos satisfy linear identities, the encoding for position pos + k is a linear function of the encoding for pos, which makes relative position trivially learnable. Modern variants (learned absolute, RoPE, ALiBi) tweak this, but the role is the same: inject ordering into a set-symmetric operation.

5. The full encoder–decoder

The Transformer wraps multi-head attention in a residual block with a position-wise feed-forward network and layer normalization, then stacks N = 6 of these on each side.

Figure 1. Encoder and decoder stacks. Solid arrows are the residual path; the dashed blue arrow feeds encoder keys and values into the decoder's cross-attention.

Anatomy of one block

Every sub-layer in the model uses the same residual-and-norm pattern:

x = x + Sublayer(LayerNorm(x))   # pre-norm variant, common in practice
# Sublayer is either MultiHeadAttention(...) or FFN(...)

The position-wise feed-forward network is two linear layers with a nonlinearity between them, applied independently to every token:

FFN(x) = max(0, x W1 + b1) W2 + b2

In the base model, the hidden dimension of the FFN is dff = 2048, four times dmodel. This block holds most of the parameters and is where the model does most of its "thinking" between attention lookups.

The decoder's two twists

  • Causal masking. During training the decoder sees the entire target sequence at once, but it must not attend to future positions or it would trivially cheat. Before the softmax in self-attention, future positions in QKT are set to −∞. After softmax those weights become zero.
  • Cross-attention. A second multi-head block per layer takes its queries from the decoder and its keys and values from the encoder's final output. This is the only channel through which source-language information enters the decoder.

6. Why this scales

The complexity story is the punchline. For sequence length n and representation width d:

Layer typeComplexity per layerSequential opsMax path length
Recurrent (LSTM)O(n · d2)O(n)O(n)
Convolutional (k)O(k · n · d2)O(1)O(logk n)
Self-attentionO(n2 · d)O(1)O(1)

Self-attention is quadratic in n and linear in d, while RNNs are linear in n and quadratic in d. For the typical regime — n in the low thousands, d in the hundreds — attention is competitive on FLOPs and dramatically better on wall-clock time because all the work is parallel matrix multiplication. Critically, every position is one hop from every other position, so gradients flow cleanly regardless of distance.

This is the property that made the modern scaling laws possible. Bigger models, longer contexts, and more data all map to operations that GPUs and TPUs already execute well. The quadratic n2 term is the only real cost — and a substantial subfield (FlashAttention, sparse attention, linear attention, state-space models) exists to chip away at it.

7. A minimal forward pass

def attention(Q, K, V, mask=None):
    # Q, K, V: (batch, heads, seq, d_k)
    scores = Q @ K.transpose(-2, -1) / sqrt(d_k)   # (b, h, n, n)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)
    weights = softmax(scores, dim=-1)
    return weights @ V                              # (b, h, n, d_v)

def encoder_layer(x):
    x = x + multihead_self_attn(layernorm(x))
    x = x + ffn(layernorm(x))
    return x

def decoder_layer(x, enc_out):
    x = x + masked_self_attn(layernorm(x))
    x = x + cross_attn(layernorm(x), enc_out, enc_out)
    x = x + ffn(layernorm(x))
    return x

That is essentially the whole model. Stack six encoder layers, six decoder layers, share the embedding matrix with the output projection, train with cross-entropy and label smoothing on a billion tokens, and you have reproduced the original WMT system. Strip the encoder and the cross-attention and you have a GPT. Keep only the encoder and add a classification head and you have a BERT. The same primitive — softmax(QKT/√dk)V — underlies all of them.

The takeaway: a Transformer replaces "carry state forward through time" with "look at everything at once, weighted by learned content similarity." Once you accept that single move, the rest of the architecture is bookkeeping: project into Q/K/V, do it in parallel heads, add positions back in, wrap in residual blocks, stack.
