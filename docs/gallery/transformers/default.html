Transformers, for People Who Already Know LSTMs

A bridge from recurrence and convolution to self-attention — the architecture behind GPT, BERT, and basically everything since 2017.

1. Why we needed something new

You already know that RNNs (LSTMs, GRUs) read a sequence one token at a time, carrying a hidden state ht forward. They work, but they have three pain points that motivated the entire transformer line of research:

  • Sequential bottleneck. Computing ht requires ht-1. You cannot parallelize along the time axis. On modern GPUs, this leaves most of the silicon idle.
  • Long-range dependencies. Information from token 1 reaches token 1000 only by squeezing through 999 hidden states. Gradients vanish, signals get overwritten, even with LSTM gating.
  • Fixed-bandwidth memory. The hidden state is a single vector. Every new token must compress what came before into the same d-dimensional slot.

Convolutions fix parallelism but introduce a different limitation: each layer only sees a local window. You stack many layers to grow the receptive field, but the path length between two distant tokens is still O(logk n) at best.

The transformer’s pitch is: make every token directly look at every other token in a single layer, in parallel. The mechanism that does this is self-attention.

2. Attention as soft, content-addressable lookup

Think of attention as a differentiable dictionary lookup. You have a query, and you want to retrieve a weighted blend of values based on how well your query matches each key.

For each token i in the input, we project its embedding xi into three vectors via learned weight matrices:

qi = xi WQ     ki = xi WK     vi = xi WV WQ, WK ∈ ℝd × dk,   WV ∈ ℝd × dv

Stack the per-token vectors into matrices Q, K, V (each row is one token). The full scaled dot-product attention is:

Attention(Q, K, V) = softmax( Q KT / √dk ) V Vaswani et al., 2017 — the central equation of the paper

Let’s read it left to right:

  1. Q KT — an n × n matrix where entry (i, j) is the dot product qi · kj. This is the unnormalized affinity of token i for token j.
  2. / √dk — the scaling. Dot products of dk-dimensional vectors have variance proportional to dk; without scaling, the softmax saturates for large dk and gradients vanish. Dividing by √dk keeps the input to softmax in a sane range.
  3. softmax(·) — applied row-wise. Now row i is a probability distribution over all tokens: how much should token i attend to each token j?
  4. · V — each output row is a weighted sum of value vectors, weighted by those attention probabilities.

Crucially, this whole computation is one big matrix multiply followed by softmax followed by another matrix multiply. The GPU loves it. And the path length between any two tokens is now exactly 1.

In self-attention — Q, K, V are all projections of the same input sequence. In cross-attention (used in the decoder), Q comes from one sequence and K, V come from another.

3. A picture is worth a few matmuls

Scaled dot-product self-attention. Every output token is a learned blend of every input value.

4. Multi-head attention: many lookups in parallel

A single attention head learns one notion of similarity. That’s limiting — a word like “bank” might want to attend differently for syntactic agreement, coreference, and topical context. The fix is brutally simple: run several attention operations in parallel with different projection matrices, then concatenate and mix.

headi = Attention(Q WiQ, K WiK, V WiV)
MultiHead(Q, K, V) = Concat(head1, …, headh) WO

Typically dmodel = 512, h = 8, and each head uses dk = dv = 64 so that the total compute roughly matches a single full-dimensional attention. The output projection WO ∈ ℝh·dv × dmodel mixes the heads back together.

Empirically, different heads specialize: some track positional offsets (“the previous token”), some track syntactic dependencies, some focus on rare content words. You don’t hand-design this — it emerges from training.

5. Positional encoding: putting order back in

There’s a problem. Attention as defined is permutation-equivariant: shuffle the input tokens and the output shuffles the same way. The model has no idea which token came first. For language (and most sequences), order matters enormously.

The fix is to add a position-dependent vector PEpos to each input embedding before the first attention layer. The original paper used fixed sinusoidal encodings:

PE(pos, 2i) = sin( pos / 100002i/dmodel )
PE(pos, 2i+1) = cos( pos / 100002i/dmodel )

Why sinusoids? Two reasons. First, every position gets a unique fingerprint across the dmodel dimensions, with low-index dimensions varying fast and high-index dimensions varying slowly — like a continuous binary clock. Second, for any fixed offset k, PEpos+k is a linear function of PEpos, which the model can exploit to reason about relative positions.

Modern variants (learned absolute embeddings, RoPE, ALiBi) have largely replaced sinusoidal encodings, but the role is identical: inject order.

6. The full encoder-decoder block

The 2017 paper’s architecture is encoder-decoder, designed for translation. Both stacks are made of identical blocks; here’s the recipe.

Encoder block (repeated N=6 times)

  1. Multi-head self-attention over the input sequence
  2. Residual connection + LayerNorm
  3. Position-wise feed-forward network (FFN): two linear layers with a ReLU/GELU between, applied independently to each position
    FFN(x) = max(0, x W1 + b1) W2 + b2
  4. Residual connection + LayerNorm

Decoder block (also N=6)

  1. Masked multi-head self-attention over outputs generated so far. The mask zeros out attention to future positions (sets their scores to −∞ before softmax) so the model can’t cheat by peeking ahead.
  2. Residual + LayerNorm
  3. Cross-attention: Q comes from the decoder, but K and V come from the encoder’s final output. This is how the decoder “reads” the source sentence.
  4. Residual + LayerNorm
  5. Position-wise FFN
  6. Residual + LayerNorm

A final linear layer projects each decoder output to vocabulary logits, followed by softmax for next-token probabilities. Training uses teacher forcing with cross-entropy loss; inference uses autoregressive decoding (greedy, beam search, or sampling).

The encoder reads the source; the decoder generates the target one token at a time, attending to its own prefix and to the encoder’s output.
Variants — Modern models often drop half the architecture. BERT is encoder-only (bidirectional self-attention, trained with masked language modeling). GPT is decoder-only (causal self-attention, trained on next-token prediction). T5 keeps the full encoder-decoder.

7. Why transformers scale so well

Compare the three architectures on three axes:

PropertyRNN / LSTM1D CNNSelf-Attention
Sequential ops per layerO(n)O(1)O(1)
Max path length between tokensO(n)O(logk n)O(1)
Compute per layerO(n · d2)O(k · n · d2)O(n2 · d)

Transformers trade quadratic compute in sequence length for two things you actually want on a GPU: full parallelism across positions, and direct connectivity between any two tokens. For sequences where n < d (a common regime), the per-layer compute is actually competitive with an RNN, while the wall-clock time is far lower because everything happens in parallel.

The deeper reason transformers became the substrate for the scaling era:

  • Uniform compute graph. A transformer is mostly matmuls of fixed shape. Compilers, kernels (FlashAttention, fused MHA), and accelerators can be optimized hard against a small, stable surface.
  • No recurrence in training. The entire forward and backward pass over a sequence is one giant batched operation. You can shard across data, sequence, tensor, and pipeline dimensions cleanly.
  • Smooth scaling laws. Loss decreases as a power law in parameters, data, and compute (Kaplan 2020, Chinchilla 2022). RNNs never showed this kind of predictable, monotonic improvement at scale.
  • One architecture, many modalities. Swap the tokenizer and the same transformer works on text, code, images (ViT), audio, protein sequences, and actions.

8. The honest caveats

  • The O(n2) attention cost is real and motivates a whole zoo of efficient variants: sparse attention, linear attention, state-space models (Mamba), sliding-window + global attention (Longformer), and KV-cache compression tricks for inference.
  • Transformers are data-hungry. With small datasets, CNNs and even MLPs can outperform them on vision tasks; inductive biases matter when you can’t throw 100B tokens at the problem.
  • Positional encoding is still an active research area. Sinusoids extrapolate badly past the training context length; RoPE, ALiBi, and YaRN all try to fix this with varying success.

The one-paragraph version

A transformer replaces recurrence with self-attention: every token directly looks up information from every other token via softmax(QKT/√dk)V, in parallel. Multi-head attention runs several of these lookups with different projections to capture different relations. Position is reintroduced via additive positional encodings since attention itself is order-blind. Stack these into encoder and decoder blocks, with residual connections, LayerNorm, and position-wise feed-forward layers, and you get an architecture that maps cleanly onto GPUs, allows direct long-range information flow in O(1) layers, and scales predictably with parameters and data — which is why it ate the field.

Reference: Vaswani et al., Attention Is All You Need, NeurIPS 2017.
