The Transformer, for the
LSTM-fluent.

You already know recurrence, gates, and convolutional receptive fields. This note is the bridge: what self-attention computes, why Q·Kᵀ works, and what changed once we stopped reading sequences one token at a time.

§ 01
why

The RNN bottleneck nobody talks about until it bites.

An LSTM reads a sequence one step at a time, folding everything seen so far into a single hidden vector ht. That recurrence is also the ceiling.

PropertyRNN / LSTMTransformer
parallelism Sequential. Step t waits for t−1. All positions in parallel. One matmul per layer.
path length O(n) hops between token 1 and token n. O(1). Every position can attend to every other.
gradient flow Vanishes/explodes through long chains. Residual + LayerNorm, depth not length.
state size Fixed d-dim hidden. Everything must fit. Stores full sequence; pays O(n²) memory for it.
inductive bias Strong: locality and order, baked in. Weak. Order comes from positional encoding only.

CNNs fix the parallelism issue but trade it for a finite receptive field — you stack layers to widen it, and long-range dependencies still travel through many hops. Both architectures share the same defect: information has to walk across the sequence to get where it needs to go.

The reframe. What if every output token could just look at every input token directly, weighting them by relevance? That is the whole idea. Everything else is bookkeeping.
§ 02
attn

Attention as soft, content-based lookup.

Think of a Python dict. You have a query, you compare it to keys, you return the matching value. Attention is the differentiable version: instead of one exact match, you return a softmax-weighted average over all values, weighted by how well their keys match the query.

Each input vector x is projected three ways:

Projections (learned, per layer)
Q = X · W_Q what am I looking for? K = X · W_K what do I contain? V = X · W_V what do I return if matched?

Same input, three different lenses. The matching itself is a single matmul; the softmax turns scores into a distribution.

Scaled dot-product attention
Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V

Why divide by √dk?

If Q and K entries have variance 1 and dimension dk, their dot product has variance dk. Large variance pushes softmax into saturation — one logit dominates, gradients vanish. Dividing by √dk keeps the pre-softmax logits at unit scale. A one-line fix with a real effect on training stability.

Read the matrix shapes once and you have it.

The [n,n] matrix is the attention map — row i, column j is "how much does token i attend to token j." This is the matrix you visualize when papers show those striped attention plots.

§ 03
heads

Multi-head: one attention is not enough.

A single attention layer collapses every relationship into one weighted average. But "the cat that the dog chased ran away" has at least two relationships you want to track at once — subject-verb (cat → ran), and object-of-relative (cat → chased). One softmax cannot represent both simultaneously without averaging them.

The fix is embarrassingly simple. Split dmodel into h subspaces, run attention independently in each, concatenate. Each head specializes — some attend to syntax, some to coreference, some to position.

Multi-head attention
head_i = Attention( X·W_Q^i, X·W_K^i, X·W_V^i ) MultiHead(X) = Concat(head_1, ..., head_h) · W_O where d_k = d_v = d_model / h (typically d_model=512, h=8, d_k=64)

Note the parameter accounting: h heads of width dmodel/h cost the same as one head of width dmodel. You get specialization for free.

§ 04
pos

Positional encoding: putting order back in.

Attention is a permutation-equivariant operation. Shuffle the input tokens, the output tokens shuffle the same way — but the content of each output is identical. To a transformer with no positional signal, "dog bites man" and "man bites dog" are the same set.

The original paper adds a fixed sinusoidal signal to each input embedding before the first layer:

Sinusoidal positional encoding
PE(pos, 2i) = sin( pos / 10000^(2i/d_model) ) PE(pos, 2i+1) = cos( pos / 10000^(2i/d_model) )

Each dimension is a sinusoid at a different wavelength — from short (~2 tokens) to long (~10,000). The key property: PE(pos+k) is a linear function of PE(pos), so the model can learn relative offsets as easy rotations rather than memorizing absolute indices. Modern variants (learned absolute, RoPE, ALiBi) refine this, but the principle stands: order is a feature you inject, not a property of the architecture.

§ 05
shape

The encoder–decoder stack, in one picture.

The original Transformer is an encoder–decoder for translation. The encoder builds a contextualized representation of the source; the decoder generates the target one token at a time, attending both to its own past and to the encoder output.

The three flavors of attention in this picture.

  • Encoder self-attention. Q, K, V all from the source. Bidirectional — every source token sees every other.
  • Decoder masked self-attention. Q, K, V from the target. Future positions are masked out (set to −∞ before softmax) so generation stays causal.
  • Cross-attention. Q comes from the decoder, K and V from the encoder output. This is the wire that lets the decoder actually use the source.

Around every sublayer: a residual connection and LayerNorm. That is what makes deep stacks trainable — gradients have a highway.

§ 06
scale

Why transformers scaled when nothing else did.

Three properties, all consequences of the design above, compound into the scaling story:

  1. Hardware fit. Every operation in a forward pass is either a matmul or an elementwise op. GPUs and TPUs are matmul machines. An RNN's sequential dependency leaves silicon idle; a transformer saturates it.
  2. Optimization regularity. Residuals + LayerNorm + Adam + warmup is a recipe that just keeps working as you stack more layers. No exploding hidden states, no vanishing recurrent gradients, no architecture surgery per task.
  3. Data efficiency at scale. The weak inductive bias is a feature in the high-data regime. The model is not assuming locality, so given enough data it learns whatever long-range structure actually exists — code, proteins, audio, images. One architecture, many modalities.
The bill. Attention is O(n²) in sequence length, in both compute and memory. Everything since 2017 — sparse attention, linear attention, FlashAttention, sliding windows, state-space models, KV-cache tricks — is the field paying down that bill while keeping the rest.

What to take from this note.

If you remember one thing: attention replaces the recurrent walk with a one-shot, content-addressed lookup over the entire sequence. Multi-head gives you parallel relationships, positional encoding gives you order back, the encoder-decoder split gives you a place to put the source. Everything else — the LayerNorms, the residuals, the warmup schedule — is the engineering that makes a stack of these things trainable at a billion parameters.
