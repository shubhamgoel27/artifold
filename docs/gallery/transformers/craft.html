Attention,
Mechanised.
A working guide to Transformers.

For the reader who knows what an LSTM cell does, who has hand-rolled a 3×3 conv, and now wants to understand why the field abandoned both — and what replaced them.

§ 01

What recurrence cost us

Premise

An LSTM processes a sequence of length n in n sequential steps. Each hidden state ht depends on ht-1. This is fine in isolation. It is catastrophic at scale.

The implication you have probably internalised but not quantified: training cannot be parallelised across the time axis. A GPU with thousands of cores must wait on a chain of matrix multiplications whose length equals the sequence. Doubling sequence length doubles wall-clock time even on a machine that could in principle do all of it at once.

The second implication is subtler. Information from position i reaches position j by being copied through every cell in between. Gradients shrink, gates leak, and by the time a sentence is 200 tokens long the model has practically forgotten its subject. LSTMs patched this; they did not solve it.

Convolutions parallelise — every output position computes independently — but pay a different price. A single conv layer of kernel k connects a position only to its k neighbours. To bridge two ends of a length-n sequence you need a stack O(logk n) deep. The path between far-apart tokens is short in layers but never zero.

OperationSequential opsMax path lengthPer-layer cost
Recurrent (LSTM)O(n)O(n)O(n · d²)
ConvolutionalO(1)O(logk n)O(k · n · d²)
Self-attentionO(1)O(1)O(n² · d)

The trade is now visible. Self-attention buys constant path length and full parallelism — and pays in n². The rest of this note is about whether that bargain was a good one (it was) and how the mechanism is actually built.

§ 02

Attention as differentiable lookup

Mechanism

Think of a Python dict. You hand it a key and it returns the matching value. Query, Key, Value — the names are not metaphorical, they are literally that operation made continuous.

Each token in the input is projected through three learned weight matrices to produce three vectors: a query (what am I looking for?), a key (what do I advertise?), and a value (what do I actually contribute if asked?). Three projections, one input.

Q = XWQ
queries — one per input position, dim dk
K = XWK
keys — same shape as queries
V = XWV
values — dim dv, often equal to dk

A token's query is dotted against every other token's key. High dot product means "this key looks like what I want." That score, after softmax-normalising into a distribution, weights the values. The output is a weighted sum.

Attention(Q, K, V) = softmax(Q K⊤√dk) V
eq. 1

The √dk denominator is not cosmetic. With high-dimensional keys, raw dot products grow with dk, pushing softmax into a regime of vanishing gradients. The scale factor keeps the variance of pre-softmax logits roughly constant in dk. Skip it and training is materially worse — Vaswani's ablation showed this.

Why this is not just RNN attention Bahdanau attention (2014) used attention as a glance from a decoder back at an encoder. The 2017 move is to use attention everywhere, including within a single sequence on itself — self-attention — and to remove the recurrence underneath it entirely. The mechanism is older than the architecture; the architecture is what changed.
§ 03

Multiple heads, multiple subspaces

Refinement

One attention computation produces one weighted view of the sequence. A single head must compromise: capture syntactic agreement, or coreference, or topical similarity — but not all three at once, because they prefer different projections of the input.

The fix is structurally trivial. Run h attention operations in parallel, each with its own WQ, WK, WV projecting into a smaller dmodel / h-dimensional subspace. Concatenate the outputs. Apply one more linear projection. Done.

MultiHead(Q, K, V) = Concat(head1, …, headh) WO
eq. 2

where each headi = Attention(QWQi, KWKi, VWVi). The total parameter count is the same as a single full-width attention — you have not added capacity, you have partitioned it. Each head is free to specialise.

Visualisations on trained models confirm the intuition. Some heads track grammatical dependencies. Some track positional offsets (attend to i – 1). Some appear to do nothing useful and can be pruned. The architecture does not require specialisation; it permits it.

§ 04

The full stack, drawn once

Architecture

An encoder layer is: multi-head self-attention, then a position-wise feed-forward network, with residual connections and layer-norm wrapped around each. Stack N = 6. The decoder adds a second attention block in the middle that attends from decoder positions to encoder outputs (cross-attention), and masks its own self-attention so position t cannot peek at t + 1.

The position-wise FFN is two linear layers with a ReLU between them, applied identically to every position: FFN(x) = max(0, xW1 + b1)W2 + b2. Inner dimension is typically 4× the model dimension (2048 vs. 512 in the base model). It does most of the parameter-counting work but none of the cross-position mixing — that is attention's job exclusively.

§ 05

Positional encoding — why the model needs to be told

Constraint

Self-attention is order-invariant by construction. Permute the input tokens and the output set is the same set, just permuted. An LSTM cannot help but encode order — recurrence is order. A pure attention stack has to be told.

The original paper adds a fixed sinusoidal signal to each token embedding before the first layer:

PE(pos, 2i) = sin(pos / 100002i/d)
eq. 3a
PE(pos, 2i+1) = cos(pos / 100002i/d)
eq. 3b

Each dimension of the embedding traces a sinusoid of a different wavelength — the lowest dimensions oscillate fast, the highest oscillate slowly across the whole sequence. The construction has a useful linear-algebraic property: any fixed offset k corresponds to a linear transformation of the encoding, so the model can in principle learn to attend by relative position.

Modern variants (learned absolute, ALiBi, RoPE, relative bias) have largely replaced sinusoidal PE in production systems, but the underlying issue is the same: ordering must enter somewhere, because attention itself refuses to provide it.

§ 06

Why this scales — and where it doesn't

Implication

The hardware story is the centre of gravity. Attention's heavy operation is a batched matrix multiply: Q K⊤ is an n × n matrix produced from two n × d inputs, which is exactly what GPUs and TPUs eat for breakfast. There is no sequential dependency to wait on — every position's output is computed simultaneously. Training throughput rises nearly linearly with accelerator count, which is the precondition for everything that followed: scaling laws, foundation models, all of it.

The bill comes due in memory. The attention matrix is O(n²) in both compute and storage. A 32k-token context is a billion-entry matrix per head per layer. This is why context length is a primary research axis: FlashAttention reorganises the computation to never materialise the full matrix; sparse-attention variants drop most of it; linear-attention approximations replace the softmax with kernels that admit O(n) decomposition. None of these are settled.

The one-sentence summary A Transformer replaces sequential recurrence with parallel, content-addressed lookup, pays the bill in quadratic context cost, and in doing so makes the entire stack a series of dense matrix multiplications — the operation modern hardware is built to perform.

If you have read this far and the architecture still feels like a list of components rather than a single object, the move is to implement it. A clean PyTorch encoder layer is around 60 lines. Once you have hand-written the einsum for scaled dot-product attention and watched the n × n matrix print to your terminal, the abstraction collapses into something concrete.
