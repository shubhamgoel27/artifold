§1 — Why recurrence broke01 / 07

An LSTM processes a sequence one timestep at a time. Each hidden state ht depends on ht−1, so the computation is intrinsically serial. Two pathologies follow.

FailureMechanismConsequence
Path lengthSignal between token i and token j traverses |i−j| nonlinear gates.Gradients vanish or distort; long-range structure is lost.
Serial computeStep t cannot start until step t−1 finishes.GPU utilization is poor; wall-clock scales with sequence length.
Fixed bottleneckSeq2seq compresses the source into one final hidden vector.Decoder must reconstruct everything from a single point.

Convolutions parallelize across positions but their receptive field grows only as O(logk n) with stacked kernels of width k. A direct edge between any two positions still requires depth.

The Transformer removes both constraints by replacing recurrent and convolutional mixing with a single primitive — scaled dot-product attention — that runs in parallel across all positions and connects any pair in one layer.

§2 — Scaled dot-product attention02 / 07

Each token emits three projections of its embedding: a query q (what am I looking for?), a key k (what do I offer?), and a value v (what do I deliver if matched?). Attention is a content-addressed soft lookup: every query reads from every key, weighted by similarity.

Stack token queries into a matrix Q ∈ ℝn×dk, keys into K ∈ ℝn×dk, and values into V ∈ ℝn×dv. Then:

Attention(Q, K, V) = softmax( Q K⊤ dk ) V
(2.1)

The product QK⊤ is an n×n matrix of raw compatibility scores. Softmax over each row turns it into a probability distribution. Multiplying by V yields a length-n sequence of mixed values — every output position is a convex combination of every input position.

The 1/√dk scaling is not cosmetic. For large dk the dot products grow with variance ~dk, pushing softmax into saturated regions where its gradient is near zero. Scaling restores well-conditioned gradients.

§3 — Multi-head projection03 / 07

One attention map can only express one relation per query — say, "match my subject's verb." Real language has many. Multi-head attention runs h attention operations in parallel on lower-dimensional projections, then concatenates.

headi = Attention(QWiQ, KWiK, VWiV)
(3.1)
MultiHead(Q,K,V) = Concat(head1, …, headh) WO
(3.2)

Each WiQ,K,V ∈ ℝdmodel×dk with dk = dmodel/h, so total compute is the same as one full-width head. The output projection WO linearly mixes the concatenated heads back to dmodel. In the base model: dmodel = 512, h = 8, dk = dv = 64.

Empirically, different heads specialize on different syntactic and semantic relations — coreference, agreement, positional offsets — without ever being told to.

§4 — Positional encoding04 / 07

Attention as defined is permutation-invariant: shuffle the rows of Q, K, V identically and the output shuffles the same way. Language is not permutation-invariant. Position must be injected explicitly.

The original paper adds a deterministic sinusoidal signal to the input embeddings:

PE(pos, 2i) = sin( pos / 100002i/dmodel )
(4.1)
PE(pos, 2i+1) = cos( pos / 100002i/dmodel )
(4.2)

Each dimension is a sinusoid of a different wavelength, geometrically spaced from 2π to 10000·2π. Because sin(a+b) and cos(a+b) are linear combinations of sin a, cos a, sin b, cos b, the model can learn to attend to relative offsets by a fixed linear transform of position vectors.

Learned positional embeddings work about as well. Sinusoidal encodings were chosen because they extrapolate — in principle — to sequence lengths unseen during training.

§5 — Encoder–decoder stack05 / 07

The full architecture is a stack of N identical encoder blocks feeding a stack of N identical decoder blocks. Each block wraps its sublayers in residual connections and layer normalization.

Per-block sublayer formula

x' = LayerNorm( x + Sublayer(x) )
(5.1)

Three attention variants appear in the diagram. They differ only in where Q, K, V come from and whether a mask is applied:

VariantQ fromK, V fromMask
Encoder self-attnsource tokenssource tokensnone
Decoder self-attntarget tokenstarget tokenscausal (upper triangle = −∞)
Cross-attntarget tokensencoder output znone

The causal mask is what makes generation autoregressive: position t cannot attend to position t+1, so the decoder can be trained with teacher forcing in parallel — one forward pass produces a logit for every output position simultaneously, but each is conditioned only on its predecessors.

§6 — Why it scales06 / 07

Three properties stacked together explain why Transformers absorb compute and data so much better than recurrent or convolutional alternatives.

PropertySelf-attentionRecurrentConvolutional
Complexity / layerO(n²·d)O(n·d²)O(k·n·d²)
Sequential opsO(1)O(n)O(1)
Max path lengthO(1)O(n)O(logk n)

The O(1) sequential count is the load-bearing entry. It means a Transformer forward pass is one big batch of matrix multiplies — exactly the workload modern accelerators were built for. Doubling the GPU count roughly halves wall-clock; doubling RNN GPUs does almost nothing per step.

The O(1) path length means any two positions exchange information in a single layer. Stacking N layers gives N rounds of arbitrary-distance mixing rather than N rounds of local mixing.

The quadratic n² term is the cost paid for these gains, and the open research front: sparse attention, linear attention, state-space models, and KV-caching all attempt to soften that n² without surrendering parallelism or path length.

§7 — Reference forward pass07 / 07

# single encoder block, batched
def encoder_block(x):                 # x: [B, n, d]
    h = multi_head_attn(x, x, x)    # Q=K=V=x
    x = layer_norm(x + h)              # residual
    h = ffn(x)                          # 2-layer MLP
    x = layer_norm(x + h)
    return x

def decoder_block(y, z, mask):
    h = multi_head_attn(y, y, y, mask)  # causal
    y = layer_norm(y + h)
    h = multi_head_attn(y, z, z)        # cross
    y = layer_norm(y + h)
    h = ffn(y)
    y = layer_norm(y + h)
    return y

What to notice

Every line is either a matmul, a softmax, or an elementwise op. There is no loop over time. The only sequence-dependent operation — the causal mask in the decoder — is implemented as adding −∞ to the upper triangle of the attention logits before softmax.

Training vs inference

At training time, the full target sequence is fed to the decoder in one pass under the causal mask. At inference, generation is autoregressive: sample yt, append, re-run. The KV-cache (out of scope here) stores past keys and values so each new step costs O(t·d) rather than O(t²·d).
