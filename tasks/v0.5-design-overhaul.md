# Folio v0.5 — design overhaul (post-design-review)

## How it started
User: "the design still looks pretty meh to me, the theme/refresh buttons feel
outdated and not in sync with the page."

## How it was reviewed
Spawned 3 parallel subagents with 10 screenshots:
- **Senior designer critique** → /tmp/folio_designer_review.md
- **Best-in-class patterns research** (Linear/Raycast/Vercel/Notion) → /tmp/folio_patterns_research.md
- **Iconography + microinteractions spec** → /tmp/folio_microinteractions.md

All three independently picked **Lucide** as the icon system. All three flagged
the topbar as the worst offender ("four button styles in 200px"). All three
recommended the sun↔moon morph as the highest-leverage single change.

## What shipped (Tiers 1 + 2 + 3 — "the works")

### T1 — Foundation
- **Inline Lucide-style SVGs** (no library dep) for: search, plus, refresh,
  list, grid, arrow-up-right, x, share, copy, check, download, theme-morph.
  Each 1.75 stroke-width, currentColor, ~16px viewBox. Stored in `ICON` map,
  injected into empty-shell buttons via `_initIcons()` at startup.
- **Unified `.icon-btn`** class — 28x28 ghost square, hover = bg-shift,
  active = scale 0.92, focus-visible = accent outline. Replaces 4 different
  ad-hoc button styles in topbar / detail header / view toggle.
- **Custom tooltip system** (~35 lines JS): delegated listener,
  `data-tip="..."` attribute, optional `data-kbd="⌘K"` for shortcut hints,
  360ms hover delay, auto-flip if near bottom of viewport, single shared
  DOM node. Native `title=` attrs are slow + unstyled.
- **Sun↔moon morph** — single SVG with `<circle>` + 8 ray `<line>`s + a
  mask that "bites" a crescent. Driven by `[data-theme="dark"]` selector
  rotating + scaling rays + sliding the mask. ~420ms. `prefers-reduced-motion`
  respected.
- **Refresh spin** — `.spinning` class adds `animation: spin 0.8s linear infinite`.

### T2 — Composition
- **Share publicly CTA demoted** from blue-primary button in detail action
  bar → ghost `icon-btn` in detail header (alongside open-in-tab + close).
  Designer's point: sharing isn't the product, browsing is — the only blue
  button shouldn't be share. Action bar now has Copy CSS + Copy skeleton
  on left, Open on right.
- **Shared state**: when artifact has been published, the share icon
  swaps to a green checkmark, tooltip changes to "Shared — click to copy URL",
  click copies instead of re-publishing.
- **Press states + focus rings** added to all interactive elements (subtle,
  via `:active { transform: scale(0.92) }` and `:focus-visible { outline }`).

### T3 — Real ⌘K command palette
- Was: "focuses the search input"
- Now: opens centered modal with **two sections**:
  - **Actions** (always shown at top when no query): Toggle theme,
    Switch to grid view, Switch to list view, Rescan library, Import a URL…
  - **Recent artifacts** (top 6 by mtime, or fuzzy-matched when typing)
- Each row: icon + label + sub (Action / dir-name)
- Section labels visually separate the two when no query is typed
- When query is present: scores actions + artifacts together, top 9 by fuzzy score
- Enter executes action OR opens artifact preview
- This is the Linear/Raycast/Vercel pattern — the patterns-research agent
  called it "the single biggest 'this feels like a modern tool' gap" in v0.4.

## Not changed (intentionally)
- **Theme toggle stayed in topbar.** Patterns agent recommended hiding
  it (Linear/Vercel/Notion don't show one). User explicitly chose to keep
  it visible since the workflow involves testing artifacts in both themes.
- **Empty state** and **mobile drawer toggle** — still on the v0.6 list.

## Sources baked into the work
- Lucide icon library (ISC license, ~0.5KB/icon)
- Refactoring UI's principle that hierarchy comes from weight/color first,
  size last — applied throughout: brand mark is the only filled blue thing
- Linear's icon-button system pattern (ghost square, hover bg-shift)
- Raycast's command-palette structure (actions + content unified)
- Microinteraction agent's sun↔moon morph spec verbatim
