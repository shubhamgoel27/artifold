# Folio v0.2 — Intent inference + source auto-detect

## The reframe
Don't capture prompts (mostly unrecoverable). Capture **intent** —
what the artifact *accomplishes* — derived from the artifact itself
via one LLM call. Cached forever by content SHA1.

## What lands
- `folio/intent.py` — LLM call returns `{intent, topic[], audience, voice}`.
  Default model: claude-haiku-4-5 (~$0.003/call). Skipped silently if no
  ANTHROPIC_API_KEY. Async with semaphore for parallel inference.
- `folio/detect.py` — pattern-based fingerprinting of source tool
  (Claude Artifacts / ChatGPT Canvas / v0 / Lovable / Bolt / Gemini)
  from HTML markers. Conservative; won't false-positive.
- Provenance schema additions: `intent`, `topic`, `audience`, `voice`,
  `intent_source` ("inferred"|"embedded"|"user"), `detection_source`.
- Scan integrates: on each file, if no `tool` set, run detect; if no
  `intent` set and intent-enabled, run inference. Persists to provenance.
- CLI: `folio scan --intent / --no-intent`, `folio analyze` (re-runs
  intent on everything), `folio enable-intent` (sets config flag).
- Config: `enable_intent: bool`, `intent_model: str`, `intent_concurrency: int`.
- Cost guard: opt-in via config. First-run nag if key present but flag off.
- UI: card snippet replaced with inferred `intent` when present. Subtle
  "auto" marker for inferred-vs-declared metadata.

## Out of scope
- `folio import <url>`, version dropdown, Claude Code hook
  (next chunk; they all build on this foundation)
- Local-model fallback (Ollama) — deferred
- Smarter detection (CSS class fingerprints) — basic patterns for now

## Course correction mid-build
User flagged: "minimal and neat, not bloated." Refactored so the LLM layer
is **opt-in only** — pulled `anthropic` out of required deps into an
`[intent]` extra. Core Folio stays fully local, zero LLM, no API key.
The intent code path is intact for users who explicitly install the extra +
set the key + pass `--intent` or set `enable_intent: true` in config.

## What landed
- [x] `folio/detect.py` — pattern-based fingerprinting (v0 / Lovable / Bolt /
      Claude / ChatGPT / Gemini / Cursor) + `<meta name="folio:*">` extraction
      for zero-LLM provenance. Pure regex, no network.
- [x] `folio/intent.py` — LLM module (only loaded when extra installed).
      Async with semaphore, cached by SHA1, returns `intent/topic/audience/voice`.
      Lazy-imports anthropic so absence is graceful.
- [x] Scan integrates: detection runs on every file (free); intent inference
      only when enabled + key + extra present. Provenance auto-enriched.
- [x] CLI: `folio scan --intent / --no-intent` flags; helpful "skipping"
      message when --intent passed but extra/key missing.
- [x] Config: `enable_intent`, `intent_model`, `intent_concurrency` (default off).
- [x] `pyproject.toml`: `[project.optional-dependencies] intent = ["anthropic>=0.40"]`.
- [x] README: documents the minimal-core / opt-in-AI split.
- [x] Template: card snippet prefers `intent` when present, falls back to
      first-paragraph snippet (so even unanalyzed cards stay readable).

## Verification
- `folio scan --no-shoot` (default): clean 15 projects, no network calls.
- `folio scan --intent` without key: prints helpful skip note, scans
  normally without LLM calls.
- Synthetic file with `<meta name="folio:intent" content="...">` +
  v0.dev URL → auto-detection set `tool=v0`, embedded meta filled
  `intent` + `intent_source=embedded`. Zero API calls.
- `folio info` on annotated file shows full provenance.
