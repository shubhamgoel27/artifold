# Folio v0.3 — version dropdown, import, share

## What shipped this session

- Options show "v2 · May 25, 2026" formatted entries (latest first)
- Changing the select swaps the iframe `src` to that version's URL,

### `folio import <url>`
- New module `folio/importer.py`, new CLI subcommand
- Private chat URLs (claude.ai/chat, chatgpt.com/c, gemini app) rejected
  with a clear pointer to `folio link` for manually-saved HTML
- Public URLs fetched via Playwright (handles client-rendered SPAs),

  on first use; persisted as `drop_dir` in config)
- Auto-populates provenance: `source` URL + `tool` inferred from hostname
- Smart filename: derived from page `<title>`, sanitized, dedup-numbered
- Verified: `folio import https://example.com --name folio-import-test`
  → file saved, provenance linked, drop dir auto-bootstrapped ✅
- Verified: private chat URL → helpful skip message, no fetch ✅

### `folio share` — public URL for any artifact
- New module `folio/share.py`, new CLI subcommand
- Default backend: GitHub Pages via `gh` CLI (most devs already have it authed)
- Uses one repo per user: `<user>/folio-share`. Creates it on first share,
  enables GH Pages on main, bootstraps a minimal index.
- Each artifact → `<short-sha-8-chars>.html` in that repo. Idempotent
  (same content = same URL, no re-push)
- Records each share into provenance as `shares: [{host, url, id, published_at}]`
- Auto-copies URL to clipboard on mac (pbcopy) / linux (xclip|xsel)
- `folio share --list` shows all shares; `folio share --revoke <id>` deletes

### Verified live, end-to-end
- `folio share ~/work/dobble-math.html` → created `shubhamgoel27/folio-share`,
  pushed 035db93c.html, GH Pages built in ~28s

### Design call: GitHub Pages over CF Pages or Folio-hosted
Cost analysis showed:
- CF Pages free tier covers up to ~10k users at $0 infra; AWS at scale = $400/yr
- **Real cost of hosted-by-Folio = operational, not infra**: DMCA, abuse,
  moderation, security incidents → 1-2 hrs/wk forever
- GH Pages uses existing user auth, zero new accounts, durable, free forever
- Cloudflare Pages = next-best opt-in (planned v0.3-beta) for users without `gh`
- Folio-hosted relay rejected (compromises "no money" stance + abuse surface)

## Out of scope (v0.3-beta / v0.4)
- Cloudflare Pages backend for users without `gh` (one-time token wizard)
- `share.folio.dev/<id>` Folio-branded URL via Cloudflare Worker rewriter
  (low-cost middle ground: domain + Worker = $15/yr, no hosting cost)
- "Diff between versions" view in preview pane
- `folio share --private` (would require GH Pro for private Pages)

      install hints (brew/apt/dnf/winget).
