# Folio v0.4 — UI redesign per /craft principles

## Why
The v0.3 dashboard violated several of the very design principles I just
shipped into the `/craft` skill:
- Aurora gradient orbs (anti-slop #11)
- Glassmorphism panels (anti-slop #7)
- Big symmetric shadows on cards (against principle #6)
- Border + shadow + glass + rounded-2xl all stacked (anti-slop #8)
- Decorative everything; no information density
- Three CLI features (share, import, design fingerprint) had **no UI surface**

It looked like a marketing landing page, not a working dashboard.

## What shipped

### Layout (Linear/Stripe-style, asymmetric)
- 48px topbar: brand + search (cmd+k hint) + `+ Import` + theme + rescan
- 220px sidebar: facets (View, Category, Tool, Date), each row with count
- Main: filter bar + scrollable list/grid + result count
- 28px status bar: doctor status + meta
- Slide-out detail pane on the right (max 880px / 60vw)

### Design tokens redo (from /craft principles)
- Greyscale ramp + ONE accent (blue). No decorative color anywhere.
- Hairline borders OR background shift, never both
- No glass blur, no aurora, no big symmetric shadows
- Display tracking on wordmark (`-0.02em`)
- Type scale: 11/11.5/12/12.5/13/13.5/14/15 (≤ 6 distinct sizes per view)
- Hierarchy via weight + color, not size

### List view (default — for power users)
- Single row per artifact: title + category dot + dir + date + intent snippet
- Right side: tool pill, version mini, shared indicator, hover actions
- 16+ artifacts visible per screen at standard zoom

### Grid view (toggleable — for visual browsing)
- 240px+ cards (denser than v0.3's 310px)
- Thumbnail + corner tool badge + minimal body
- Same shared/version indicators

### Detail pane — three tabs
- **Preview**: full iframe of the artifact
- **Provenance**: tool/source/model/intent/topics/audience/voice/prompt/
  tags/path/sha1/shares (all surfaced as a kv list with clickable URLs)
- **Design**: palette swatch bar (clickable to copy hex), font pills
  (rendered in their actual font), mood flags (gradient/glass/etc),
  radius/density/CSS-size stats

### Action bar in detail pane
- `Share publicly` (primary) — calls POST /share, returns URL, copies to clipboard
- `Copy CSS template` — GETs /designs/<sha>?format=css, copies
- `Copy skeleton` — GETs /designs/<sha>?format=skeleton, copies

### Sidebar facets (multi-filter)
- View: All / Shared / Local only
- Category: from existing data (multi-select)
- Tool: claude / chatgpt / v0 / lovable / bolt / gemini / cursor / manual / unlabeled (multi-select)
- Date: Last 7 days / Last 30 days
- All filters compose

### Topbar
- `+ Import` opens modal with URL input — POSTs to /import
- Theme toggle (no decorative emoji, just sun/moon)
- Rescan button with spinner
- Search with ⌘K hint

### Empty state
- Shown when 0 projects: three quick-cards (add a folder / import a URL /
  install /craft skill)

### Server endpoints added
- **POST /share** `{path}` → runs share.share_via_gh, returns `{ok, url}`
- **POST /import** `{url}` → runs importer.import_url, returns `{ok, path}`
- **GET /designs/<sha>?format=css|skeleton|template** → returns design dump
  with path-scoped security check (must be under a configured root)

### Inline row actions (hover-reveal)
- ↑ (share) for unshared rows; ⎘ (copy URL) for shared rows
- ↗ open in new tab

### Live updates
- After share/import, server triggers a background re-scan; SSE pushes
  `event: updated`; client refetches data.json + re-renders without reload.
  If detail pane is open, it stays open with refreshed data.

### Responsive
- Sidebar collapses below 760px (basic — needs a drawer trigger button, deferred)
- Detail pane goes full-width on mobile

## What was dropped
- All aurora/gradient orb background
- All glassmorphism / backdrop-filter blur
- Gradient text on wordmark (replaced with solid brand mark + word)
- Big symmetric card shadows
- Decorative tool icons / category emoji
- Card preview hover scale (excessive animation)

## Bug caught
- `.live` CSS class collision: row's `.share-state.live` inherited
  `position:fixed; top:12; right:18; display:none` from the toast notification
  class. Renamed to `.is-shared` to disambiguate.

## Verified
- 16 projects render in list view, fully visible on 1500x1000 viewport
- Detail pane: all 3 tabs work, palette swatches clickable to copy
- Server endpoints: `/share` idempotent (returns same URL), `/import`
  rejects private chat URLs with helpful error, `/designs/<sha>?format=...`
  returns 7,106 chars of CSS for dobble + clean skeleton
- Both themes (dark default + light) verified at hi-DPI
- Grid view toggle works
