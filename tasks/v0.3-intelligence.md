# Folio v0.3 — interactive init + lightweight design intelligence

## Design call: extract + delegate, don't embed

The intelligence layer is built around a single principle: **Folio extracts
the reusable design materials; whichever LLM the user prefers does the
generation.** No vector DBs, no embeddings, no Folio-side LLM cost.

Per artifact (at scan time, pure regex, ~ms):
  - **palette** — top 6 colors (frequency-ranked, hex-normalized)
  - **fonts** — first family from each font-family declaration
  - **design tokens** — CSS custom properties from `:root`
  - **mood flags** — gradient / glass / themed / animated / shadowed
  - **avg radius** — quick "soft vs sharp" signal
  - **density / sizes** — body text length, CSS size

Stored in provenance under `design`, keyed by content SHA1 (so file moves
preserve the fingerprint).

For generation: `folio designs <id> --template` dumps the full CSS + a
content-free body skeleton. User pastes into Claude / Cursor / their LLM
of choice and says "make a new report with this style." The LLM gets exact,
copyable design materials — much better in-context signal than any
abstract fingerprint we'd invent.

## What shipped

### Interactive `folio init` wizard
- TTY-guarded (skipped on pipes / `--non-interactive`)
- Lists existing roots if any; offers to add more
- Scans `~/Downloads`, `~/Documents`, `~/Desktop` for candidates, shows
  html-file counts per dir to give a confidence signal
- After picking root: offers initial scan + dashboard auto-open
- Zero-to-working-dashboard in <60s for new users

### `folio/design.py`
- `extract(html)` → compact JSON fingerprint
- `as_template(html)` → ready-to-paste CSS + body skeleton
- Integrated into `_enrich_provenance` (runs free during every scan)

### CLI: `folio designs`
- `folio designs` → table of all artifacts with hash/name/palette/flags
- `folio designs <id>` → JSON fingerprint for one artifact
- `folio designs <id> --css` → just the `<style>` contents
- `folio designs <id> --skeleton` → body markup with text stripped
- `folio designs <id> --template` → CSS + skeleton bundled

## Verification
- All 16 cached artifacts got design fingerprints on rescan
- Listing shows correct palettes + flags per artifact
  (dobble: dark gradient anim; Eye Health: minimal/serif; etc.)
- `--css` dump: 283 lines for dobble (real, reusable)
- `--skeleton` dump: `<div.wrap><h1><p.lede>...` structure preserved,
  text removed — exactly what an LLM needs to mimic layout

## Out of scope (later)
- A bundled Claude Skill `/visualize` that wraps `folio designs` calls
  for in-Claude generation (just markdown + tool spec; ships next)
- `folio generate --like <id> --topic ...` direct API call (opt-in via
  `[intent]` extra) — when user wants single-command generation rather
  than copy-paste flow
- Diff between similar artifacts ("what changed in style v1 → v2")
- Mood/voice LLM enrichment to augment the regex signals

## The whole pipeline now
```
Generate (any AI tool)        →  artifact lands in a scanned root
folio                          →  auto-indexes, extracts design, screenshots
folio designs <id> --template  →  paste into Claude for next report
folio share <new-report>       →  public URL for the result
folio import <url>             →  pull a shared artifact back in
```
