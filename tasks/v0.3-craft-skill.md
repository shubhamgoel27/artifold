# Folio v0.3 — `/craft` Claude Code skill

## What this ships

A Claude Code skill — markdown file installed at
`~/.claude/skills/craft/SKILL.md` — that activates whenever the user asks
for a report / dashboard / one-pager / explainer / tracker / guide /
itinerary. It does three things at once:

1. **Real design knowledge** — 12 opinionated principles distilled from
   Refactoring UI, Vercel/Geist, and Linear's design philosophy. Each
   principle is grounded in a specific source (cited inline).
2. **Style transfer** — if the user references a past artifact ("like
   dobble"), the skill calls `folio designs <id> --template` to load
   the CSS + body skeleton as a style baseline.
3. **Anti-slop variation by default** — if no reference is given, the
   skill calls `folio designs` to see the user's library and deliberately
   varies from their past patterns. Their library becomes a portfolio of
   distinct artifacts, not 20 takes on one template.

Plus a 15-item anti-pattern list (the specific signatures of AI-default
HTML to avoid) and a 12-item self-check to run before delivering.

## How it was built

- **Research delegated to a subagent** with a tight brief: distill real
  sources into 8-12 grounded principles + an anti-pattern list.
  Returned 8 web fetches + 3 corroborating searches.
- Sources used: Refactoring UI (landing, free color chapter, "7 Practical
  Tips" article, "Top 20 Key Points"), Linear method intro, Vercel/Geist
  via Seedflip's breakdown, TheAdpharm + MindStudio AI-slop critiques.
- Research output: `tasks/craft-design-research.md` (~140 lines, fully
  cited, opinionated)
- Skill prompt distilled from research into ~200 lines at
  `folio/skills/craft/SKILL.md`

## CLI: `folio install-skill`
- Copies the bundled SKILL.md into `~/.claude/skills/craft/`
- Idempotent (warns + suggests `--force` if already installed)
- `--dest <path>` for non-default location
- Prints example `/craft …` prompts to help user get started

## Packaging
- `folio/skills/craft/SKILL.md` ships as package data
- Subpackage `folio.skills.craft` (with `__init__.py`) makes it loadable
  via `importlib.resources.files("folio.skills.craft")`
- pyproject `[tool.hatch.build.targets.wheel.force-include]` ensures
  the .md file ends up in the wheel

- Multiple skills (e.g. `/folio-tweak` to iterate on an existing artifact,
  `/folio-mix` to blend two styles) — defer; ship the one well first
