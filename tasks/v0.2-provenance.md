# Folio v0.2 — Provenance + Versions

Goal: every artifact remembers where it came from, and related iterations
(`-v2`, `-v3`, ` (1)`) collapse into a versioned timeline instead of three
disconnected cards.

## Data design

Provenance store: `<cache>/provenance.json` — keyed by **content SHA1**
(so files survive moves/renames). Each entry:

```json
{
  "source":  "https://claude.ai/share/xxxx",
  "tool":    "claude" | "chatgpt" | "v0" | "lovable" | "bolt" | "manual" | null,
  "model":   "claude-opus-4-7" | null,
  "prompt":  "<original prompt>" | null,
  "tags":    [],
  "notes":   "",
  "added_at":"<iso>"
}
```

Schema versioned via top-level `"version": 1` so we can migrate later.

## Version detection

New `VERSION_RE = ([-_ ]v\d+|[-_ ]\(\d+\))$` (case-insensitive).
Strip to get "logical stem"; group files in a dir by logical stem.
Files in the same logical-stem group with different versions → **one project,
N versions**, primary = highest version. Variants (`-print`, `-onepage`)
continue to attach to a primary as before.

## CLI

- `folio link <file> [--source URL] [--tool claude] [--model NAME] [--prompt TEXT] [--tag X]`
- `folio info <file>` — print provenance + project membership
- (Deferred to v0.2-beta: `folio import <url>` — fetch + auto-annotate)

## UI

- Card badge (top-right of thumb): tool icon + short label (Claude / ChatGPT / v0)
- Card meta: "v3 · 4 versions" when versioned
- Preview pane: version dropdown when versioned; switching swaps iframe src
